# 安卓应急收银 App 设计

日期：2026-07-05

## 目标

为阿根廷华人超市制作一个原生 Android 手机收银 App，用于停电或电脑收银系统不可用时继续营业。App 离线运行，不同步、不修改电脑收银软件数据库，只作为内部应急账本。停电结束后，通过导出的 Excel/CSV 在电脑收银系统里补录或对账。

所有设计文档、分析文件、后续 Android 工程和生成文件都放在当前工作文件夹：

`E:\手机收银软件开发`

## 范围

第一版包含：

- 原生 Android App，兼容 Android 10+。
- 单台手机独立使用，本机保存商品库和销售记录。
- 支持导入鸣盛收银系统拷贝出来的 SQLite `.db` 商品数据库。
- 支持导入 Excel `.xlsx` 商品表作为备用路径。
- 支持手机相机扫码、手动输入条码、商品名搜索。
- 支持收银、改数量、删行、改价、行优惠、整单优惠、结账保存。
- 支持查看交易明细和每日总账。
- 支持按日期导出 Excel/CSV。
- 支持商品库版本记录和回退上一版。

第一版不包含：

- 不做正式发票或税务票据。
- 不直接修改电脑收银数据库。
- 不做多手机实时同步。
- 不连接电子秤。
- 不连接打印机，不出小票。
- 不做一单多种付款方式混合支付。
- 不凭空实现当前数据库里没有数据支持的复杂促销。

## 平台与兼容性

技术路线为原生 Android，推荐 Kotlin。最低系统版本为 Android 10。

兼容策略：

- 使用标准 Android API，不依赖某个手机品牌的私有能力。
- 重点兼容小米澎湃/MIUI、荣耀 MagicOS、OPPO ColorOS、vivo OriginOS、三星等常见安卓派生系统。
- 文件导入使用 Android 系统文件选择器。
- 扫码使用标准相机权限。
- 离线数据保存在 App 本地数据库中。

## 模块化设计约束

后续实现必须按模块化方式设计，避免把导入、商品、优惠、收银、账本、导出逻辑写在同一个大文件或同一个界面里。

建议模块边界：

- `app`：Android 入口、导航、权限、主题、语言切换。
- `catalog`：商品查询、商品库版本、商品分类、条码文本处理。
- `importer`：`.db` 和 `.xlsx` 导入适配器，只负责把外部文件转换成内部商品模型。
- `pricing`：价格计算、自动优惠、手动改价、行优惠、整单优惠。
- `checkout`：购物车、结账流程、付款方式、作废流程。
- `ledger`：销售记录、交易明细、每日总账。
- `exporter`：Excel/CSV 导出。
- `settings`：语言、当前商品库版本等本机设置。

关键约束：

- UI 层不直接读取鸣盛原始数据库表，只调用内部仓库或用例。
- 促销和价格计算必须集中在 `pricing`，不要散落在界面代码里。
- `.db` 和 `.xlsx` 导入必须作为两个独立适配器，后续新增导入格式时不影响收银逻辑。
- 销售记录保存结账当时的商品名称、价格和优惠结果，不能依赖后续商品库状态。
- 新增功能时优先新增独立模块或扩展接口，不修改无关模块。
- 单个文件过大时要拆分，界面、业务规则、数据访问要分开。
- 每个核心模块要能单独测试，特别是 `pricing`、`importer`、`ledger` 和 `exporter`。

## 商品导入

导入流程：

1. 用户从手机文件选择器选择 `.db` 或 `.xlsx`。
2. App 校验文件是否可读。
3. App 解析商品数量、条码、名称、售价、分类、单位、优惠字段。
4. App 显示导入结果摘要。
5. 用户确认后生成新的商品库版本。

每次导入保留版本记录：

- 导入时间。
- 来源文件名。
- 商品数量。
- 可识别促销数量。
- 导入状态。

用户可以回退到上一版商品库，防止导错旧库或坏文件。

## 价格与促销规则

价格和促销只照搬当前可读数据库中能读取到的数据，以及鸣盛手册中能与数据库字段对应的规则。

第一版确定支持：

- 普通零售价。
- 数据库中的优惠价和起购数量。
- 达到起购数量后，整行全部数量按优惠单价计算。

示例：某商品起购数量为 2，优惠单价为 1499。购买 3 个时，3 个都按 1499 计算。

当前规则优先级：

1. 手动改价。
2. 手动行优惠。
3. 数据库自动优惠价。
4. 普通零售价。

手动改价或手动优惠会覆盖数据库自动优惠，不叠加。如果同一行同时存在手动改价和手动行优惠，先把手动改价作为新的基准单价，再计算手动行优惠。

手册中存在但当前可读数据库没有对应促销数据的规则，例如买送、绑定、量打折、量定价、总额优惠等，第一版只预留数据结构和计算接口，等以后拿到可读促销数据后再接入。

## 收银流程

主流程：

1. 打开收银界面。
2. 扫码、手动输入条码，或搜索商品名。
3. 商品加入购物车。
4. 用户可修改购物车。
5. 选择付款方式。
6. 保存销售单。

商品行支持：

- 修改数量。
- 删除。
- 手动改价格。
- 百分比优惠。
- 固定金额优惠。

整单支持：

- 百分比优惠。
- 固定金额优惠。

同一笔销售每次只使用一种整单优惠类型，百分比优惠和固定金额优惠不同时叠加。

付款方式：

- 现金。
- Mercado Pago。
- 借记卡。
- 信用卡。
- transferencia。

第一版一单只选择一种付款方式。

## 未找到商品与手动价格

扫码或输入条码未找到商品时，弹出“未找到商品”弹窗。

弹窗行为：

- 可以通过 Android 返回键或返回手势关闭。
- 可以通过弹窗自带取消键关闭。
- 提供“手动输入价格”按钮。

手动输入价格后生成一条临时商品行：

- 分类固定为 `almacen`。
- 商品名可显示为手动商品或 `almacen`。
- 记录输入价格、数量、时间和销售单号。

称重或散装商品不按重量计算。因为店内称重商品会提前贴价格标签，收银时只需要扫码价格标签或手动输入价格。

## 销售记录

每一笔销售保存完整明细。

销售单头记录：

- 销售单号。
- 创建时间。
- 付款方式。
- 原始总额。
- 整单优惠类型。
- 整单优惠金额。
- 实收总额。
- 状态：正常或已作废。

销售明细行记录：

- 商品条码。
- 商品名称。
- 分类。
- 数量。
- 原始单价。
- 自动优惠单价。
- 手动改价。
- 行优惠类型。
- 行优惠金额。
- 最终单价。
- 行小计。
- 是否手动价格商品。

已经保存的销售允许作废，但不删除记录。作废单不计入正常每日总账，并可在作废记录中查看。

## 查看与对账

App 提供三个管理入口。

交易明细：

- 按日期查看销售单列表。
- 可进入单笔交易详情页。
- 详情页显示商品、数量、原价、优惠、付款方式、总额、状态。

每日总账：

- 按日期汇总销售额。
- 显示订单数。
- 显示作废单数量和作废金额。
- 按付款方式汇总现金、Mercado Pago、借记卡、信用卡、transferencia。

导出：

- 支持按日期范围导出 Excel/CSV。
- 导出销售单汇总表。
- 导出销售明细表。
- 用于电脑收银系统补录和对账。

## 语言

界面支持中文和西班牙语切换。默认可以先使用中文，设置中可切换到西语。

核心按钮和页面名称应有对应翻译，例如：

- 收银 / Caja
- 商品 / Productos
- 交易明细 / Ventas
- 每日总账 / Resumen diario
- 导入商品 / Importar productos

## 本地数据模型

建议本地数据库至少包含这些表：

- `product_versions`：商品库版本。
- `products`：当前可用商品。
- `promotion_rules`：从商品库读取到的可用促销规则。
- `sales`：销售单头。
- `sale_items`：销售明细行。
- `app_settings`：语言、当前商品库版本等设置。

商品导入时先写入新版本，确认成功后切换当前版本。销售记录保存商品当时的名称、价格和优惠结果，避免以后商品库更新后影响旧销售明细。

## 错误处理

导入错误：

- 文件打不开：提示文件不可读。
- 表结构不匹配：提示不是支持的鸣盛商品库。
- 条码为空或价格异常：导入摘要中显示异常数量。
- 导入失败：保留当前商品库不变。

收银错误：

- 未找到商品：显示弹窗，并允许手动输入价格。
- 价格或数量为空：禁止加入购物车。
- 总额小于零：禁止结账。

导出错误：

- 文件保存失败：提示重新选择位置。
- 没有记录：导出空结果前提示用户。
