# 手机端同步连接界面修复设计

## 范围

仅修改 `ImportScreen` 的手动连接弹窗、连接结果展示和异步回调管理。保留现有 IP、端口、Token 输入方式以及 `ComputerSyncService` 调用边界；不修改 HTTP 协议、配置校验、同步业务或电脑端代码。

## 界面行为

- 电脑 IP 输入框保持局域网示例 `192.168.1.35`，不引导用户填写 `127.0.0.1`。
- 点击“保存并测试连接”后，按钮禁用并显示轻量测试状态，防止重复请求。
- 成功时关闭弹窗、刷新页面，并展示电脑工具版本、已连接 IP 和端口。`ComputerSyncService` 返回已验证的轻量健康结果，页面不读取 JSON。
- 失败时保持弹窗与三项输入不变，恢复提交按钮，便于直接修正配置。
- 离开导入页后，尚未完成的测试不再触发弹窗或刷新页面。页面维护 `AtomicInteger` 任务代数和可用标记；视图脱离时递增代数、标记不可用并中断仍在运行的任务线程，所有主线程回调先验证代数。

## 错误展示

`ImportScreen` 仅通过 `ComputerSyncException.reason()` 读取稳定的 `ComputerSyncFailureReason`，不检查异常消息文本。页面将超时、连接拒绝、Token 错误、无效配置或 IP、响应无效、明文 HTTP 被阻止及其他 HTTP/未知错误映射为对应的用户提示；错误详情不包含 Token 或完整请求 URL。

## 边界与验证

- 页面不直接创建 HTTP 连接，只调用 `ComputerSyncService`。
- 将连接成功和失败文案收敛到小型 UI helper，后台任务只负责执行与回到主线程。
- 验证现有同步 smoke test、Debug APK 构建和最终合并 Manifest；代码检查确认页面未按异常文本分类，且没有电脑同步扫码入口。
